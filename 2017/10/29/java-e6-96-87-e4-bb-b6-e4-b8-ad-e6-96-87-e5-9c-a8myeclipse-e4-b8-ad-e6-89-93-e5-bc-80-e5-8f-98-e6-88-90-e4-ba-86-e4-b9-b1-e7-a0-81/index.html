java文件中文在MyEclipse中打开变成了乱码

我在使用Myeclipse时出现一个问题，由于有几个类是在一个压缩包中想要使用，于是我就直接将它复制到了Myeclipse中了。然后打开时就出现了以下的情况：

中文乱码是因为编码格式不一致导致的。

　　进入Eclipse,导入一个项目工程,如果项目文件的编码与你的工具编码不一致 将会造成乱码。
　　如果要使插件开发应用能有更好的国际化支持，能够最大程度的支持中文输出，则最好使 Java文件使用UTF-8编码。

但是在这里，我们不设置为UTF-8格式，因为我试过了，这个格式还是乱码 :arrow: :arrow: ，那我们应该设置为什么格式呢？又怎么设置？ 我们设置为 gbk 格式的，然后就可以解决乱码问题了。

修改默认编码：
　　在菜单导航栏上Window–>Preferences 打开”首选项”对话框，左侧导航树，导航到 General–>Workspace。导航栏window–>preferences
打开”首选项”对话框，左侧导航树，导航到 Genera–>Content Types
　　右边找到要修改的文件的类型,我这边以*.java为例
　　在下面的Default encoding，输入框中输入gbk->Update->OK

windows—-properties—–general——-workspace——-text file encoding —-Text—-java source file 改成gbk，然后update，就可以了 不介意就给个好评吧~ 还有问题的话，可以评论的哦~
